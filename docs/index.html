Mesh Simplification

Abstract

The above image was created from our final project — Mesh Simplification. Mesh simplification is an important tool during real-time rendering. Objects in world space that are far away or are not essential features in a scene probably don't need a large amount of detail and can be simplified using mesh simplification, thereby saving memory and rendering time. The high-level overview of our mesh simplification algorithm is that we first compute the cost of collapsing all the edges, and sort them in the order of least to highest cost. The cost of edge contraction is computed using a quadric error system that was specified in Michael Garland's dissertation on Quadric-Based Polygonal Surface Simplification. We then collapse the edge with the least cost at each iteration, and keep on repeating this process for a desired level of simplification. This level can be modified using an in-built factor variable inside our codebase.

Technical Approach

Here is the full layout of our algorithm:

  1. Compute the quadric error for all vertices. This will help us determine which vertices we will need to eventually remove. The system for computing this quadric error will be described in the following sections.
  2. At each iteration, we compute the optimal contraction target v for each valid pair (v1, v2). Define the cost of contracting a pair as the dot product of v and ((Q1 + Q2)v), where Q1 and Q2 are the quadric error matrices of vertex v1 and v2.
  3. Calculate the pair that has the lowest cost of edge contraction.
  4. Iteratively contract the minimum cost pair, and update costs of all other pairs involving the new contracted vertex.

Edge Collapse/Contraction

The first step was writing edge contraction. This was done by calculating the new

optimal point between the two vertices. There are two ways of computing this optimal point, one which is introduced in Garland’s dissertation (where the optimal v = (-1) * inverse of A * b) and the other is our own customized solution. Our customized solution computes the midpoint of v1 and v2 while considering the difference in the degree of each vertex. The difference between the two approaches will be clearly seen in the demo below. Our optimal vertex equation is the following:

(v1->degree() * v1->position + v2->degree() * v2->position) / (v1->degree() + v2->degree());

We then create a new vertex V with this optimal position, and reassign the necessary neighbors’s attributes to the new optimal vertex. One of the challenges we faced during this step was making the decision of using the old vertex versus creating a completely new one. After much debugging, we decided that creating a new vertex and reassigning the necessary parts was rather simpler than modifying the position of one of the vertices and reassigning all the parts connected to it.

Downsample

Downsampling is very similar to upsampling run in reverse. We use both error quadrics and edge collapsing to downsample an entire mesh. We set a parameter m which tells the downsample method how many edges we wish to collapse. This variable allows for higher structure quality at the cost of more downsamples required. First, we calculate the error quadrics for all edges in the mesh. Then, we find the minimum cost edge(v1, v2) and call the edge collapse method on it. We update the cost for all edges that touch v1. This process repeats the desired number of times, which is determined by the initial parameter m.

Challenges

There were several major problems we encountered during the course of this project. The first issue was actually selecting a project topic to start with. We initially chose a controllable 3D robot skeleton that would mimic a human’s walking pattern, but had to change topics once we realized there was not enough of a technical component to it. Once we decided on mesh simplification, it was straightforward to begin adding the edge collapse method to the existing project 2 framework.

After edge collapse was mostly correct, we moved on to the error quadrics matrix calculations. However, the matrix4x4 class in C++ did not work when performing basic matrix-vector multiplications. Even when checking the dimensions of both operands matched, it would throw an error or simply return the wrong result. Ultimately, we had to perform individual element-wise multiplications to get the final error quadric for a vertex pair.

The next and arguably most frustrating issue occurred when trying to use a priority queue as the min heap in the algorithm. At first, we used the standard C++ priority_queue implementation, but unlike Java/Python implementations, we were unable to update the priorities of existing halfedges after the initial construction. So we decided to switch to the provided mutable_priority_queue class, but it turns out this was an even bigger failure. This implementation broke the entire algorithm, so we had to switch back to the original standard library implementation. We initially worked around updating by clearing the PQ after each edge collapse, recalculating all the costs for required vertex pairs, and adding all the vertex pairs back into the PQ. However, we realized that this practically eliminated the need for a PQ in the first place, as we were iterating through all the edges each time and selecting only the minimum cost edge. So we got rid of the PQ and created a method to find the minimum cost edge (that had not already been collapsed) for any given mesh.

The biggest overarching problem that was completely unavoidable was the time zone difference. Having only a 5-6 hour window that all three of us were awake simultaneously made it really difficult to work together on the same task. This led to splitting up the project into different sections and most of the core implementation for any method was done by a single person, with everyone helping to debug afterwards.

Lessons Learned

Although the time zone difference made it nearly impossible to work on the same part simultaneously, it allowed us to efficiently break up the work into manageable tasks. Each person worked on one of the three major parts of the algorithm, and having each part implemented by a different person allowed us to learn how to properly write code such that it could be used and understood by anyone else. Additionally, we learned more about the halfedge mesh data structure, because we had to delete mesh elements in this algorithm, whereas project 2 only added or changed elements. References

The research paper that we found most helpful is Surface Simplification Using Quadric Error Metrics by Garland and Heckbert. In this paper, Garland and Heckbert use what is known as ‘edge contraction’ to merge the vertices on each end of the edge while maintaining the structure of the remaining triangles. They iterate this on a pair of vertices that compose an edge, or on a pair of vertices whose distance is less than the threshold that they define, and finally compute error quadric matrices. More information on their work can be found here: Surface Simplification Using Quadric Error Metrics
